## Decisions

### executionLoop exit on PAUSED
When the loop detects `run.status === 'PAUSED'`, it **returns** (not sleep+loop). This allows `resumeRun()` to call `executeRun()` from PAUSED status and cleanly restart the loop. Avoids two concurrent loops.

```
pauseRun()  → DB: PAUSED, _pausedRunIds.add(runId)
               → executionLoop detects PAUSED → return (clean exit)
resumeRun() → _pausedRunIds.delete(runId)
               → executeRun(PAUSED) → transitions PAUSED→RUNNING → new loop
```

### Deadlock detection
When `getReadyNodes()` returns `[]` and no nodes have `status='RUNNING'`, the loop breaks. This handles the case where downstream nodes are `PENDING` but their deps are `FAILED`/`INTERRUPTED`/`DEADLOCKED` (they can never become ready).

Without this fix: infinite `await sleep(100ms)` loop → OOM after timeout.

### InMemoryRunStore.seedNode auto-id
Seeded nodes MUST have a unique `id` (the DB primary key). Without it, all `node.update({ where: { id: undefined } })` calls match the same node (the first in the array, since `undefined === undefined`). This caused:
- transitionNode(n2, RUNNING) → actually updated n1 to RUNNING
- n1 appeared RUNNING forever → infinite loop → OOM at ~35s
`seedNode()` now auto-assigns `crypto.randomUUID()` if `id` is missing.

### AgentRunnerFn abstraction
The executor accepts `agentRunner: AgentRunnerFn` via constructor injection. In prod (T1.6): real `IAgentRunner`. In tests: `MockAgentRunner` (inline arrow function). This avoids `jest.mock()` complexity.

### babel-preset-current-node-syntax is incompatible with plugin-transform-modules-commonjs
`babel-preset-current-node-syntax` intentionally disables module transformation (it's a syntax-only preset). Adding `plugin-transform-modules-commonjs` on top causes conflicts and OOM during jest worker startup. Solution: use `@babel/preset-env { targets: { node: 'current' } }` (standard recommended setup).

## Issues encountered & fixed

| Issue | Root cause | Fix |
|---|---|---|
| jest OOM (4GB+) | Infinite loop + babel config conflict | (1) Fixed loop + (2) switched to `@babel/preset-env` |
| Two concurrent executionLoops | `resumeRun` was calling `executeRun` while original loop still ran | PAUSED → `return` (not sleep); resumeRun re-launches cleanly |
| Deadlock on FAILED deps | n3=PENDING blocks when n2=FAILED; loop never found ready nodes | `if no ready && no running → break` |
| `seedNode` id collision | `node.update({ where: { id: undefined } })` always matched row 0 | `seedNode` auto-assigns UUID |
| `babel-preset-current-node-syntax` OOM | Conflicts with `plugin-transform-modules-commonjs` | Replaced by `@babel/preset-env` |
