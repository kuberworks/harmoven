## 3. Technical decisions

### 3.1 Optional injection, not mandatory

`IProjectEventBus` is an **optional** constructor param with `undefined` default.
If `_eventBus` is absent, `_emit()` is a pure no-op. This keeps all existing
tests green without modification — no test injects a bus, and the existence of
events in the bus is not tested at the unit level (that belongs to integration tests).

### 3.2 Fire-and-forget with error logging

`_emit()` calls `this._eventBus.emit()` via `void ... .catch(...)`.
A bus failure (PgNotify disconnect, InMemoryEventBus subscriber crash) must
never propagate into the execution loop. The worst case is a missed SSE frame —
not a failed run.

### 3.3 `_runProjectIds` cache

`project_id` is fetched once in `executeRun()` and stored in a Map
keyed by `runId`. This avoids a DB fetch per transition while keeping
`transitionRun()` and `transitionNode()` signatures unchanged.
The cache entry is deleted in the `finally` block of `executeRun()`.

### 3.4 Factory: lazy require + null sentinel

In the factory, `eventBus` defaults to the `projectEventBus` singleton
(lazy `require()` to avoid circular import at module level).
Passing `eventBus: null` explicitly disables bus injection (tests).
Passing `eventBus: myBus` overrides with a custom/mock bus (future integration tests).

### 3.5 HumanGate events via utility (not executor)

HumanGates are created by HTTP route handlers (T3.2), not by the executor.
`lib/events/emit-gate.ts` provides a single function `emitHumanGateEvent()`
that emits both:
- `human_gate` RunSSEEvent → received by run-level SSE (filtered by `stream:gates`)
- `gate_opened` ProjectLifecycleEvent → received by project-level SSE (dashboards)

T3.2 route handlers will call this after writing the HumanGate row to the DB.

### 3.6 Fields sourced safely from run model

`run.task_input` is typed as `Json` in Prisma — handled with a null guard and
`JSON.stringify()` fallback. `run.domain_profile` may be absent from test
mock objects — handled with `?? 'unknown'` fallback.

## 5. Security

| Vector | Mitigation |
|--------|-----------|
| Bus error propagating to executor | `void ... .catch(err => console.error(...))` — never rethrows |
| Event with attacker-controlled `message` in `error` event | `message` comes from `err instanceof Error ? err.message : String(err)` — no raw user input |
| `task_summary` truncated to 120 chars | `.slice(0, 120)` — prevents oversized payloads on reconnect buffer |
