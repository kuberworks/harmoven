## Key decisions

1. **Prisma 7** — `url` moved to `prisma.config.ts` at project root. Uses `dotenv` to load `.env` since Prisma config runs before its internal env loading.

2. **Better Auth CLI vs manual merge** — CLI's jiti runtime cannot construct Prisma 7 client (requires driver adapter). Manual merge is exact equivalent: plugin schemas inspected via Node.js before writing.

3. **`better-auth.prisma` kept** — stored as `prisma/better-auth.prisma` for future CLI diff comparison when upgrading better-auth.

4. **HARMOVEN_DB_PORT env var** — docker-compose uses `${HARMOVEN_DB_PORT:-5432}` so it works out of the box on clean machines; set to 5433 in local `.env` when 5432 is taken.

5. **AuditLog immutability** — PostgreSQL `RULE` approach (runs INSTEAD OF, does nothing). Alternative would be triggers but rules are simpler for append-only tables.

6. **Node partial index deferred** — Prisma doesn't yet support `@@index([field], where: ...)`. Plain `Node_last_heartbeat_idx` created by Prisma; partial index optimization tracked for future migration.

7. **Migration history clean** — single migration `20260325171434_init`, no drift, `migrate deploy` verified on clean `harmoven_ci` database.
