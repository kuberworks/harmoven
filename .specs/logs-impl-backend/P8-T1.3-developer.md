## Decisions

### Prisma 7 — library engine removed
Prisma 7 dropped the library/binary engines entirely. `engineType = "library"` in the generator
block is silently ignored. All database access must use a driver adapter:
```typescript
const adapter = new PrismaPg({ connectionString: process.env.DATABASE_URL })
new PrismaClient({ adapter })
```

### Seed: no upsert on null FK
`@@unique([project_id, name])` with `project_id = null` cannot be targeted by Prisma's `upsert`
(composite unique with nullable FK). Replaced with `findFirst` + conditional `create`/`update`.

### RBAC design
- 27 permissions as a TypeScript union type (compile-time exhaustiveness)
- 7 built-in roles with additive inheritance (viewer → operator → … → instance_admin)
- `instance_admin` bypasses all project membership checks in `resolvePermissions()`
- IDOR prevention: `assertProjectAccess` and `assertRunAccess` return identical `ForbiddenError`
  whether the resource doesn't exist or the caller lacks access (prevents enumeration)

### Better Auth config
- `admin({ adminRole: 'instance_admin', defaultRole: 'user' })` — role stored as plain string in `user.role`
- `trustedOrigins` + `baseURL` both read from `AUTH_URL` env var
- `AUTH_SECRET` required in production (throws at startup if missing)
- `requireEmailVerification` skipped when `AUTH_SKIP_VERIFY=true` (local dev convenience)
