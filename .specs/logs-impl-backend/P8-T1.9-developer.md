## Decisions

### ILLMClient interface unchanged
Agents already used `chat()` + `stream()` from `mock-client.ts`. `DirectLLMClient`
implements the same interface — no agent changes needed.

### Tier → profile resolution
`options.model` = `'fast' | 'balanced' | 'powerful'` → `selectByTier()` finds the
first active profile matching that tier. Direct profile IDs and provider model strings
are also accepted as fallbacks.

### CometAPI = OpenAI-compatible (Amendment 71)
CometAPI is routed through the `openai` npm package with `baseURL: 'https://api.cometapi.com/v1'`
and `api_key_env: 'COMETAPI_API_KEY'`. No separate SDK needed.

### Ollama = OpenAI-compatible
Same as CometAPI — `openai` package with `baseURL: 'http://localhost:11434/v1'` and
`apiKey: 'no-key'` (Ollama ignores the key for local use).

### Gemini system prompt
Gemini's `generateContent` API uses `systemInstruction` at model construction time
(not in the messages array). `splitMessages()` extracts system-role messages and
passes them as `systemInstruction`.

### createLLMClient() guards against test usage
The factory throws in `NODE_ENV=test` — agents in tests must inject `MockLLMClient`
directly. This prevents accidental real API calls during CI.

### Integration test gating
Real API tests are skipped unless `HARMOVEN_LLM_TIER=haiku` AND `ANTHROPIC_API_KEY`
are set. Four unit tests (tier resolution, profile loading) always run.

### Multi-criteria scorer scope
`selectLlm()` implements the full Section 6 algorithm (confidentiality gate,
jurisdiction gate, context window gate, scoring). It is ready for the DAG executor
to use in production. Agents continue to use the simpler `selectByTier()`.
