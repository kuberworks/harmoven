### 2. LiteLLM Docker sidecar isolation (`docker-compose.yml`)

Replaced the commented-out LiteLLM block with a production-ready isolated sidecar:
- Image pinned by digest (`${LITELLM_DIGEST}`) — prevents tag-hijacking
- Gated behind `--profile litellm` (not started by default)
- `litellm_internal` network: `internal: true` — Docker-enforced, no external traffic
- LiteLLM receives: only `LITELLM_MASTER_KEY` + LLM provider keys
- App receives: only `LITELLM_GATEWAY_URL` (not the raw provider keys)

### 3. GPG signature verification (`lib/marketplace/install-pack.ts`)

Implemented step 2b which was explicitly deferred since T3.6+:
- `verifyGPGSignature()`: uses `execFile` (no shell), temp files in `mkdtemp` dir
- Controlled by `marketplace.security.verify_gpg_signature` in orchestrator.yaml
- Defaults to `false` — operators must import the Harmoven registry public key first
- Hash mismatch now also emitted to supply-chain-monitor (previously was silent)

### 4. Supply chain monitor (`lib/security/supply-chain-monitor.ts`)

New module providing typed wrappers for security events:
- 7 event types: `image_digest_mismatch`, `pack_signature_invalid`, `pack_hash_mismatch`,
  `mcp_skill_hash_mismatch`, `update_without_github_release`, `dependency_version_mismatch`,
  `litellm_version_drift`
- All map to `warning` or `critical` severity
- Writes to AuditLog which is append-only (PostgreSQL RULE prevents UPDATE/DELETE)
- Degrades gracefully on DB failure: logs to stderr, does not crash the process

### 5. MCP skill hash verification (`lib/bootstrap/verify-mcp-skills.ts`)

Startup integrity check for configured MCP skills:
- Validates `version` is well-formed semver
- Computes SHA-256 of `entrypoint` file
- Compares against `sha256` configured in orchestrator.yaml
- Mismatch: calls `reportMCPSkillHashMismatch()` and returns a failure result
- `strict=true` mode aborts startup on first failure

### 6. Config (`orchestrator.yaml`)

```yaml
  verify_pack_signatures: false   # enable after keyring setup

```

### 7. ESLint no-restricted-imports (`.eslintrc.json`)

**What to be honest about:**

  `scan-pack-prompts.js` with `|| true` (non-blocking). This is intentional since
  the `packs/` directory doesn't exist yet, but should be made blocking in T3.9.

## Decisions made

1. **GPG defaults to `false`**: Forcing GPG verification on from day 1 would break
   existing installs for operators who haven't set up the keyring. Documented path
   to enable it; set `true` as soon as first beta release is signed.

2. **LiteLLM Docker profile gate**: Used `profiles: [litellm]` so the sidecar is
   not started in the default `docker compose up`. Prevents ports from being allocated
   unnecessarily and makes the opt-in explicit.

3. **Supply chain monitor degrades gracefully**: Chose not to throw if AuditLog write
   fails — a monitoring failure should not cascade into a user-visible error. The
   original security violation is already being handled (GPG fail = install blocked).

4. **`verify_gpg_signature: false` vs `true` in orchestrator.yaml**: Left as false
   with a comment explaining keyring setup requirements. Setting it to `true` here
   would immediately break the marketplace for any deployment without the key imported.
